## Вопрос 1

**В чём главный недостаток «реактивной поддержки без методологии» для организации-разработчика?**

1. Нельзя прогнозировать сроки и бюджет; качество без процесса со временем деградирует  
2. Такой режим запрещён платформой 1С  
3. Он всегда быстрее Scrum  
4. Он подходит только для строительства  

---

## Вопрос 2

**Почему жёсткий «водопад» (PMBOK) на длительном ИТ-проекте часто даёт расхождение плана с реальностью?**

1. Требования на длинных проектах меняются, а последовательный план плохо это учитывает  
2. Водопад запрещает документировать этапы  
3. Водопад применим только к файловым базам  
4. В PMBOK нет этапов  

---

## Вопрос 3

**Для чего по смыслу лекции удобен Kanban в сравнении со Scrum?**

1. Для поддержки и потока задач по этапам; видно узкие места на доске  
2. Только для выгрузки `.dt`  
3. Только когда продукт полностью неизвестен заранее — Kanban обязателен  
4. Kanban заменяет справочник номенклатуры

## Вопрос 4

**Зачем в динамическом списке включать «Динамическое считывание данных» при большом справочнике и большом числе остатков?**

1. Платформа порционирует выборку по видимой области — экономия памяти  
2. Чтобы отключить запрос к виртуальной таблице  
3. Чтобы склад передавался только на клиенте  
4. Чтобы убрать права на регистр

## Вопрос 5

**Что обычно означает для виртуальной таблицы «Остатки» регистра накопления незаданный или пустой период в параметрах?**

1. Берётся актуальный остаток «на сейчас» — одно из быстрых обращений  
2. Остатки всегда нулевые  
3. Запрос к остаткам становится недоступен  
4. Период обязательно подставляется как начало года

## Вопрос 6

**Зачем вводить параметр сеанса со ссылкой на справочник «Пользователи», если есть пользователи информационной базы?**

1. Пользователь ИБ — не ссылочный тип; для реквизитов документов нужен элемент справочника  
2. Параметры сеанса могут быть только строками  
3. Справочник пользователей не может использоваться в формах  
4. Чтобы запретить журнал регистрации  

---

## Вопрос 7

**Что при переименовании реквизита документа (например, синоним «поставщик» → «покупатель») обычно сохраняется, а где главная «ловушка»?**

1. Тип ссылки и данные в базе сами по себе не ломаются; опасность — код по старому имени и объединение конфигураций  
2. Все ссылки на контрагента обнуляются  
3. Документ перестаёт проводиться без пересоздания ИБ  
4. Печатная форма обновляется автоматически без проверки  

---

## Вопрос 8

**Почему у ограниченной роли может не отображаться добавленная команда объекта?**

1. У роли нет прав на объект метаданных — добавленную команду объекта; нужно выдать просмотр  
2. Команды объектов доступны только полным правам без исключений  
3. Команды объектов не зависят от ролей  
4. Нужно отключить подсистему «Сервис»
