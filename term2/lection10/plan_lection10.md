### Блок 1: Введение в AI-агенты (15 минут)

#### 1.1 Что такое AI-агенты (5 минут)

**AI-агент** — это система с высокой степенью автономности, которая способна достигать поставленных целей, адаптируясь к нештатным ситуациям и используя итеративный подход.

**Ключевое отличие от Workflow:**
- **Workflow** — детерминированная система с четким алгоритмом действий
- **AI-агент** — автономная система, способная адаптироваться и принимать решения в процессе работы

**Аналогия:**
- **Workflow** — кофейный автомат: нажал кнопку → получил кофе
- **AI-агент** — бариста в кофейне: понимает контекст, предлагает варианты, адаптируется к ситуации

**Основные применения:**
- Customer care (служба поддержки клиентов)
- Автоматизация рутинных задач
- Работа с нестандартными ситуациями

---

#### 1.2 Ключевые свойства AI-агентов (10 минут)

**1. Автономность (Autonomy)**
- Агент не должен привлекать пользователя без необходимости
- Работает самостоятельно для достижения цели
- Главное свойство AI-агента

**2. Динамическое принятие решений**
- Решения принимаются в процессе работы
- Невозможно предсказать точный путь выполнения заранее
- Зависит от внешних факторов и контекста

**3. Real-time data investigation**
- Данные могут быть получены в процессе работы агента
- Агент использует актуальную информацию для принятия решений

**4. GPT-based reasoning**
- Использует языковые модели для рассуждений
- LLM — основа работы AI-агента

**5. Continuous learning**
- Агент должен иметь память
- Запоминает успешные решения и использует их в будущем
- Без памяти агент неэффективен

**6. Итерационный подход**
- Агент работает циклами: действие → анализ результата → корректировка
- Может повторять попытки до достижения цели
- Адаптируется на основе обратной связи

**7. Взаимодействие со средой**
- Использует инструменты (tools) для выполнения задач
- Без инструментов агент ограничен в возможностях

### Блок 2: Архитектура AI-агентов (20 минут)

#### 2.1 Компоненты AI-агента (10 минут)

**1. Input (Вход)**
- Пользовательские сообщения
- Задачи и цели от пользователя

**2. Output (Выход)**
- Результаты работы агента
- Ответы пользователю

**3. Actions (Действия)**
- Выполнение конкретных операций
- Использование инструментов

**4. Reasoning (Рассуждение)**
- Планирование действий
- Анализ ситуации
- Принятие решений

**5. Feedback Loop (Обратная связь)**
- Анализ результатов действий
- Корректировка плана
- Human-in-the-loop (при необходимости)

**Пример работы:**
- Запрос: "Принеси мне кофе"
- Reasoning: нужна кофемашина, кофе, кофемолка
- Actions: взять кофе, перемолоть, залить в кофемашину, нажать кнопку
- Результат: кофе готов

---

#### 2.2 System Message (10 минут)

**System Message** — ключевой компонент AI-агента, определяющий его поведение и логику работы.

**Назначение:**
- Определяет, как агент действует
- Описывает цели агента
- Задает правила инициализации
- Определяет, как поддерживать цикл итераций
- Указывает, какие инструменты использовать и когда

**Содержание System Message должно включать:**
1. **Описание роли агента** — кто он и что делает
2. **Цели агента** — что он должен достигать
3. **Правила работы** — как он должен действовать
4. **Описание инструментов** — какие tools доступны и когда их использовать
5. **Жизненный цикл** — как агент должен работать итеративно

**Важно:**
- Программирование AI-агента — это в первую очередь написание System Message
- Это основная часть разработки агента
- Может называться по-разному: базовый промпт, agent message, system prompt

---

### Блок 3: Инструменты (Tools) и MCP (20 минут)

#### 3.1 Что такое Tools (10 минут)

**Tools (Инструменты)** — это внешние функции, которые агент может вызывать для выполнения задач.

**Без инструментов агент ограничен:**
- Может только генерировать текст
- Не может взаимодействовать с внешним миром
- Не может выполнять практические задачи

**С инструментами агент может:**
- Искать информацию в интернете
- Работать с базами данных
- Выполнять действия в системах
- Генерировать изображения
- И многое другое

**Примеры инструментов:**
- **Веб-поиск** — поиск информации в интернете
- **RAG (Retrieval Augmented Generation)** — поиск по корпоративным знаниям
- **Генерация изображений** — создание изображений (может быть субагентом)
- **Работа с базами данных** — выполнение запросов
- **Интеграции с системами** — работа с 1С, CRM и др.

---

#### 3.2 MCP (Model Context Protocol) (10 минут)

**MCP (Model Context Protocol)** — протокол для создания инструментов, совместимых со всеми моделями и фреймворками.

**Преимущества MCP:**
- Пишете инструмент один раз — работает везде
- Не привязан к конкретной модели или фреймворку
- Стандартизированный подход

**Awesome MCP Service:**
- Существует множество готовых MCP серверов
- Маркетинг, мониторинг, обработка видео, безопасность, медиа и многое другое
- Можно комбинировать в своих агентах

**Для работы с 1С:**
- MCP Server для 1С позволяет создавать инструменты внутри системы
- Инструменты могут быть доступны как MCP серверы
- Упрощает интеграцию агентов с корпоративными системами

**Важно:**
- Не все конструкторы агентов требуют MCP
- Могут быть сторонние инструменты
- Но MCP становится стандартом индустрии

### Блок 4: ReAct подход и память (15 минут)

#### 4.1 ReAct подход (8 минут)

**ReAct (Reasoning + Acting)** — подход, при котором агент чередует рассуждения и действия.

**Цикл работы:**
1. **Reasoning** — анализ ситуации, планирование
2. **Action** — выполнение действия (вызов инструмента)
3. **Observation** — анализ результата действия
4. **Reasoning** — корректировка плана на основе результата
5. Повторение до достижения цели

**Пример:**
- Запрос: "Какая погода в Москве?"
- Reasoning: нужен инструмент для поиска погоды
- Action: вызов инструмента погоды
- Observation: получен прогноз "около нуля"
- Reasoning: нужно добавить текущую дату, вернуть результат в нормальном формате
- Action: возврат результата пользователю
- Завершение работы

**Ключевое свойство:**
- Агент должен собирать feedback
- Корректироваться на основе обратной связи
- Запоминать предпочтения пользователя

---

#### 4.2 Память агента (7 минут)

**Типы памяти:**

**1. Краткосрочная память:**
- Хранится в контексте LLM
- Часть текущего контекста
- Расширяется по мере работы
- Пока задача не закончена

**2. Долгосрочная память:**
- Сохраняется в базе данных (векторная БД)
- Успешные решения и паттерны
- Используется в будущих задачах
- Позволяет агентам учиться

**Важно:**
- Оба типа памяти полезны
- Краткосрочная обычно управляется фреймворком
- Долгосрочная тоже часто управляется фреймворком
- Но разработчик должен понимать важность памяти

**Пример использования:**
- Пользователь: "Мне не нужна погода на день, нужна на месяц"
- Агент запоминает это предпочтение
- В следующий раз сразу выдает погоду на месяц

### Блок 5: Мультиагентные системы (10 минут)

#### 5.1 Концепция мультиагентных систем

**Мультиагентная система** — система, состоящая из нескольких специализированных агентов.

**Зачем нужны:**
- Один агент не может решить все задачи одинаково хорошо
- Чем более специализированный агент, тем лучше он решает свою задачу
- Для корпоративной среды нужны разные агенты для разных задач

**Примеры задач:**
- Вопросы по зарплате → агент по кадрам
- Вопросы по серверам → агент по IT
- Вопросы по корпоративным системам → агент по системам

**Роутинг-агент:**
- Понимает запрос пользователя
- Задает уточняющие вопросы при необходимости
- Направляет запрос нужному специализированному агенту
- Работает как диспетчер в call-центре

**Уровни агентов:**
1. Агент не связан с кодом — принимает решения в системе
2. Агент управляет выполнением программы
3. Вывод агента определяет действия кода
4. Агент выполняется оперативно, определяет действия, может вызывать инструменты
5. Мультиагентная система — агенты обращаются друг к другу

### Блок 6: Кейсы применения AI-агентов (30 минут)

#### 6.1 Customer Care и поддержка (5 минут)

**Консультант по кадрам (AppleDocs):**
- Помогает с отпусками
- Анализирует стоимость отпуска в разные периоды
- Проверяет пересечения с коллегами
- Дает рекомендации по оптимальным датам
- Работает как умный помощник, а не тупой чат-бот

**Бот поддержки:**
- Индексируется база знаний
- Дается доступ к базе данных
- Создается Telegram-бот за пару дней
- Может заменить целый колл-центр
- Обрабатывает рутинные запросы

---

#### 6.2 Финансовые и бухгалтерские кейсы (10 минут)

**Банк с выпиской:**
- Агент находит банковскую выписку на сайте
- Адаптируется к изменениям интерфейса банка
- Загружает выписку в 1С
- Ставит проводки на основе назначения платежа
- Выбирает статьи ДДС исходя из знаний

**Нейроаудитор:**
- Пишет запросы к базе данных на естественном языке
- Проверяет балансовые ведомости
- Находит красное сальдо
- Рекомендует исправления
- Работает по расписанию в фоне
- Правила добавляет бухгалтер (не разработчик!)

**Агент по вводу первички:**
- Закинул документ в чат → появился в 1С
- Или скриншот → Ctrl+V в 1С → документ создан
- OCR с пониманием сущности документа
- Исправляет ошибки распознавания на основе контекста

**BI-агент:**
- Доступ ко всем отчетам компании
- Пользователь задает вопрос на естественном языке
- Агент отвечает, используя данные отчетов
- Заменяет аналитиков и отчеты

**Агент по платежам:**
- Анализирует письма от директора
- Проверяет полномочия
- Находит контрагента или создает нового
- Формирует платежку
- Выгружает в банк
- Остается только подписать

---

#### 6.3 HR и продажи (5 минут)

**HR-агент:**
- Читает резюме
- Пишет соискателям
- Назначает интервью
- Определяет соответствие по скиллам
- Может работать через RPA с HeadHunter

**Sales Agent:**
- Обогащение CRM
- Ищет информацию о лидах в интернете
- Находит профили в LinkedIn
- Создает события (дни рождения, подарки)
- Формирует персонализированные предложения
- Экономит 60% времени sales-менеджеров

**Анализатор звонков:**
- Расшифровывает записи звонков
- Добавляет расшифровку в CRM
- Оценивает качество звонка
- Фиксирует договоренности
- Создает краткую выжимку
- Может заменить супервизора отдела продаж

---

#### 6.4 Производство и логистика (5 минут)

**Управление товарными запасами:**
- Получает информацию о продажах и остатках
- Определяет необходимость подсортировки
- Планирует заказы на склад
- Анализирует динамику продаж
- Может существенно улучшить бизнес

**Система прогнозирования сбоев:**
- Анализирует логи производства
- Находит ошибки и сбои
- Автономный контроль и управление
- Регулирует процессы в реальном времени

**Управление цепочкой поставок:**
- Оптимизирует закупки
- Планирует логистику
- Определяет оптимальное время доставки
- Контроль качества продукции

#### 6.5 Дополнительные кейсы (5 минут)

**Compliance Agent:**
- Проверяет новых контрагентов
- Проверяет по санкционным спискам
- Проверяет сайты и документы
- Делает скриншоты

- Получает корпоративную почту

### Блок 7: Практическая демонстрация в N8N (25 минут)

#### 7.1 Создание простого агента (15 минут)

**Задача:** Создать агента, который получает информацию о продажах и отвечает на вопросы пользователя.

**Шаги создания:**

1. **Создание Simple AI Agent:**
   - Открыть N8N
   - Создать новый Simple AI Agent
   - Базовый вариант с чатом

2. **Настройка триггера:**
   - Chat Message — для сообщений в интерфейсе
   - Альтернативы: Webhook, Email, HTTP Request
   - Можно сделать публичным

3. **Настройка модели:**
   - Создать Credential для OpenAI
   - Вставить API ключ
   - Выбрать модель (GPT-4o mini или GPT-4)
   - Можно использовать OpenRouter

4. **Настройка памяти:**
   - Simple Memory (для начинающих)
   - Context Windows: 5 последних сообщений
   - Позволяет агенту помнить контекст

5. **Добавление инструментов:**
   - Для работы с 1С: MCP Server
   - Создать инструмент GetSales (получение продаж)
   - Выбрать отчет (например, "Валовая прибыль")
   - Опубликовать как MCP сервер

6. **Настройка System Message:**
   - Описать роль агента
   - Указать, что нужно использовать инструменты
   - Проанализировать результат инструмента
   - Ответить на русском языке

7. **Тестирование:**
   - Открыть чат
   - Задать вопрос о продажах
   - Проверить работу инструмента
   - Убедиться, что агент анализирует данные

---

#### 7.2 Публикация и использование (5 минут)

**Публикация чата:**
- Сделать чат публичным
- Получить ссылку для доступа
- Активировать workflow
- Тестировать через публичный интерфейс

**Возможности расширения:**
- Добавить больше инструментов
- Подключить другие MCP серверы
- Улучшить System Message
- Добавить обработку ошибок

---

#### 7.3 Интеграция с 1С через MCP (5 минут)

**MCP Server для 1С:**
- Используется конструктор MCP Server для 1С
- Создание инструментов внутри 1С
- Публикация как MCP сервер
- Подключение к агенту через localhost или удаленный адрес

**Важно:**
- Публикация MCP серверов желательно без аутентификации
- Отдельная публикация для безопасности
- Инструменты доступны агенту как tools

### Блок 8: Платформы и инструменты (10 минут)

#### 8.1 Популярные платформы

**N8N:**
- Изначально система автоматизации workflow
- Теперь поддерживает AI-агентов
- Визуальный интерфейс
- Простое создание агентов

**LangChain:**
- Популярный фреймворк для разработки агентов
- Кодовая среда разработки
- Гибкость и расширяемость

**LangGraph:**
- Самый популярный представитель кодовых сред
- Графовая структура агентов
- Мощный инструментарий

**Другие:**
- Cursor, Cloud Code — примеры сложных агентов
- Множество других фреймворков и платформ

---

#### 8.2 RAG как основной паттерн

**RAG (Retrieval Augmented Generation):**
- Базовый тренд в продуктах с AI-агентами
- Решение проблемы ограниченного контекста LLM
- Поиск релевантной информации и добавление в контекст
- Обязательно нужно знать

**Стек для запуска агентов:**
- Аутентификация
- Инструменты (tools)
- Доступность через разные варианты оркестрации
- Модель для агента
- Стек или облако для развертывания

### Блок 9: Тренды и перспективы (5 минут)

#### 9.1 Рост интереса к AI-агентам

**Статистика:**
- Резкий рост интереса к AI-агентам
- Промпт-инжиниринг уже не так интересен
- Fine-tuning и обучение с подкреплением тоже
- AI-агенты — новый тренд

**Применение:**
- Практически везде, где работа связана с обработкой информации
- Если работа только с компьютером — повод задуматься
- Революция уже происходит

**Рекомендация:**
- Начинать разрабатывать агентов сейчас
- Стать центром компетенции
- Возглавить революцию, а не быть замененным

## Резюме лекции (5 минут)

### Ключевые моменты:

1. **AI-агенты vs Workflow:**
   - AI-агенты — автономные системы с адаптацией
   - Workflow — детерминированные системы с четким алгоритмом
   - Агенты лучше справляются с нестандартными ситуациями

2. **Ключевые свойства:**
   - Автономность — главное свойство
   - Динамическое принятие решений
   - Итерационный подход
   - Непрерывное обучение
   - Взаимодействие со средой через инструменты

3. **Компоненты AI-агента:**
   - System Message — определяет поведение
   - Tools — инструменты для выполнения задач
   - Memory — краткосрочная и долгосрочная
   - Reasoning — рассуждения и планирование
   - Actions — выполнение действий

4. **ReAct подход:**
   - Reasoning → Action → Observation → Reasoning
   - Цикл повторяется до достижения цели
   - Агент собирает feedback и корректируется

5. **Инструменты:**
   - MCP (Model Context Protocol) — стандарт для инструментов
   - Множество готовых MCP серверов
   - Инструменты критически важны для агентов

6. **Кейсы применения:**
   - Customer care и поддержка
   - Финансовые и бухгалтерские задачи
   - HR и продажи
   - Производство и логистика
   - Практически везде, где обрабатывается информация

7. **Платформы:**
   - N8N — визуальная разработка
   - LangChain, LangGraph — кодовая разработка
   - Множество других фреймворков

8. **Практика:**
   - Создание агента в N8N достаточно простое
   - Интеграция с 1С через MCP
   - Можно быстро создать работающего агента
