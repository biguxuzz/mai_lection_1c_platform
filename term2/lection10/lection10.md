### Блок 1: Введение в AI-агенты (15 минут)

#### 1.1 Что такое AI-агенты (5 минут)

Сегодня мы разберемся, что такое AI-агенты, зачем они нужны и где применяются. На семинаре в эту субботу мы попробуем создать агента для вашей конфигурации.

**AI-агент** — это система с высокой степенью автономности, которая способна достигать поставленных целей, адаптируясь к нештатным ситуациям и используя итеративный подход.

**Ключевое отличие от Workflow:**
- **Workflow** — детерминированная система с четким алгоритмом действий
- **AI-агент** — автономная система, способная адаптироваться и принимать решения в процессе работы

**Аналогия:**
- **Workflow** — кофейный автомат: нажал кнопку → выбрал напиток → вставил деньги или приложил карточку → получил заказанное → забрал. Если не вставил деньги — до свидания. Если не выбрал что-то — ничего не получится. Каждый раз мы повторяем одни и те же действия.
- **AI-агент** — бариста в кофейне: понимает контекст, предлагает варианты, адаптируется к ситуации. Если мы подойдем к автомату и скажем "сегодня я не хочу американо", автомат ничего не сделает. Бариста же может предложить альтернативу.

**Основные применения:**
- Customer care (служба поддержки клиентов)
- Автоматизация рутинных задач
- Работа с нестандартными ситуациями

---

#### 1.2 Отличия агентов от workflow (10 минут)

**Workflow — детерминированный процесс:**

Пример: согласование документа
- В программу приходит PDF
- Проверяется правильность формата файла
- Проверяется, что это точно PDF, а не картинка
- Добавляется в карточку документов
- На этом рабочий процесс прерывается

**Проблемы workflow:**
- Если пришло задание, а файла нет — нужно заранее продумать обработку исключений
- Все возможные сценарии должны быть предусмотрены заранее
- Невозможно обработать нестандартные ситуации

**AI-агент — итеративный подход:**

**Как работает агент:**
1. Что-то происходит на входе
2. Агент анализирует, что произошло на входе
3. Пытается принять решение, что с этим делать
4. Если нужны уточнения — уточняет у пользователя
5. Цель — достичь результата через итеративный подход

**Пример работы агента:**
- Задание: нужно карточку предложить (паспорт, страховой договор)
- Пользователь забыл прислать один из документов
- Агент не прекращает работу (как в случае workflow)
- Агент говорит: "Нужен еще такой-то документ"
- Возвращается к уточнению

**Основные отличия:**

1. **Автономность:**
   - У агента есть скоуп задач, который он может решать
   - Если что-то идет не по сценарию, он возвращается к уточнению

2. **Динамическое принятие решений:**
   - В workflow все заранее продумано
   - Агент может сам подумать, что здесь что-то не достает, и повторить действие

3. **Обучаемость:**
   - Агенты основаны на больших языковых моделях
   - Дообучаются в процессе
   - История чата сохраняется
   - Удачные решения складываются в базу для последующего использования

4. **Взаимодействие с внешней средой:**
   - Агент использует инструменты (tools) для выполнения задач
   - Без инструментов агент ограничен в возможностях

### Блок 2: Архитектура и принципы работы AI-агентов (20 минут)

#### 2.1 Компоненты AI-агента (10 минут)

**Схема работы агента (по примеру HiddenFace):**

1. **Появляется запрос пользователя**
2. **Агент обдумывает** — что же ему нужно сделать, какие запустить инструменты
3. **Запускает инструменты** — использует доступные tools
4. **Возвращает результат пользователю**

**Инструменты (Tools) — это MCP-серверы:**
- В VS Code мы видели, как создаются файлики
- LLM — это просто подбор следующего слова за предыдущим, исходя из контекста
- **Tool calling** (вызов инструментов) — обязательная история для агентов

**Без инструментов агент бесполезен:**
- Чатики поддержки, которые ничего не делают — никому не нужны
- Полезный агент должен вызывать инструменты, фиксировать данные, предлагать решения

**Примеры инструментов:**
- Работа с файловой системой
- Веб-поиск
- Сохранение контекста в memory bank (локальный RAG)
- Генерация кода
- Сохранение файлов
- Использование командной строки

---

#### 2.2 System Message (10 минут)

**System Message** — ключевой компонент AI-агента, определяющий его поведение и логику работы.

**Назначение:**
- Определяет, как агент действует
- Описывает цели агента
- Задает правила работы
- Определяет, какие инструменты использовать и когда

**Содержание System Message должно включать:**
1. **Описание роли агента** — кто он и что делает
2. **Цели агента** — что он должен достигать
3. **Правила работы** — как он должен действовать
4. **Описание инструментов** — какие tools доступны и когда их использовать
5. **Жизненный цикл** — как агент должен работать итеративно

**Важно:**
- Программирование AI-агента — это в первую очередь написание System Message
- Это основная часть разработки агента
- Может называться по-разному: базовый промпт, agent message, system prompt

**Пример:**
- System Message: "Ты кинокритик фильмов 90-х годов, твоя цель — отвечать пользователю про эти фильмы"
- Описываем роль, которую выполняет агент
- Дальше работаем с этим

### Блок 3: ReAct подход и память (15 минут)

#### 3.1 ReAct подход (8 минут)

**ReAct (Reasoning + Acting)** — подход, при котором агент чередует рассуждения и действия.

**Цикл работы:**
1. **Reasoning** — анализ ситуации, планирование
2. **Action** — выполнение действия (вызов инструмента)
3. **Observation** — анализ результата действия
4. **Reasoning** — корректировка плана на основе результата
5. Повторение до достижения цели

**Пример работы:**
- Запрос: "Какая сегодня погода?"
- Reasoning: пользователь задал вопрос о погоде, судя по cookies он находится в Москве
- Action: для того чтобы узнать погоду, нужно использовать инструмент погоды
- Observation: инструмент вернул погоду (один градус ниже нуля)
- Reasoning: нужно вернуть это пользователю в человекочитаемом виде
- Action: возврат результата пользователю
- Завершение работы

**Ключевое свойство:**
- Агент должен собирать feedback
- Корректироваться на основе обратной связи
- Адаптироваться под обратную связь и дальше действовать

---

#### 3.2 Память агента (7 минут)

**Типы памяти:**

**1. Краткосрочная память:**
- Хранится в виде текста (самая простая вариация)
- Ограничение: например, последние 5 сообщений
- Вся история чата сохраняется
- Используется для понимания контекста текущего разговора

**2. Долгосрочная память:**
- В более сложных вариантах — все выкладывается в RAG
- Успешные решения фиксируются в долгосрочную память
- При следующем подобном запросе экономит токены на размышления
- Запрос похожий — размышлять уже не нужно, используем такие-то инструменты

**Важно:**
- Оба типа памяти полезны
- Краткосрочная для контекста текущего разговора
- Долгосрочная для обучения и оптимизации

**Пример использования:**
- Пользователь: "Мне не нужна погода на день, нужна на месяц"
- Агент запоминает это предпочтение
- В следующий раз сразу выдает погоду на месяц

### Блок 4: Мультиагентные системы и уровни агентов (10 минут)

#### 4.1 Мультиагентные системы

**Концепция:**
- Чем более специализированный агент, тем лучше он решает свою задачу
- В корпоративной среде нужна мультиагентская система

**Примеры специализации:**
- Пользователь на сайте хочет заказать справку 2 НДФЛ → вызывается агент, который работает с кадровой системой
- Пользователь спрашивает про регламенты работы в компании → агент лезет в RAG и забирает информацию

**Роутинг-агент:**
- Понимает запрос пользователя
- Направляет запрос нужному специализированному агенту
- Работает как диспетчер

---

#### 4.2 Уровни агентов

**Уровень 1:** Агент не связан с кодом — просто чатик на сайте, особо ничего не делает, какой-либо пользы не приносит

**Уровень 2:** Агент управляет каким-нибудь одним инструментом через API

**Уровень 3:** Агент использует несколькими инструментами

**Уровень 4:** Агент сам понимает, что можно завершить работу

**Уровень 5:** Мультиагентская система

**Для наших задач:**
- Скорее всего, будем делать что-то на третьем-четвертом уровне
- Агенты, которые работают в конфигурации 1С

### Блок 5: Кейсы применения AI-агентов (30 минут)

#### 5.1 Customer Care и поддержка (5 минут)

**Служба поддержки — самый успешный кейс:**
- Большой поток обращений, которые обрабатываются примерно по шаблону
- Работа в службе поддержки — рутинная, мозг отключается через некоторое время
- Все вопросы накапливаются, ты уже ночью будешь отвечать как LLM

**Пример из X5 Retail:**
- Отдел из 60 человек превратился в 3 человека
- Оставшиеся обслуживают агенты, которые собирают данные и делают отчеты
- Автоматизация рутинных задач

**Консультант по кадрам (AppleDocs):**
- Помогает с отпусками
- Анализирует стоимость отпуска в разные периоды
- Проверяет пересечения с коллегами
- Дает рекомендации по оптимальным датам
- Работает как умный помощник, а не тупой чат-бот

---

#### 5.2 Финансовые и бухгалтерские кейсы (10 минут)

**Банк с выпиской:**
- Агент находит банковскую выписку на сайте банка
- Адаптируется к изменениям интерфейса банка
- Загружает выписку в 1С
- Ставит проводки на основе назначения платежа
- Выбирает статьи ДДС исходя из знаний
- Бухгалтер больше не нужен для загрузки выписки

**Нейроаудитор:**
- Пишет запросы к базе данных на естественном языке
- Проверяет балансовые ведомости
- Находит красное сальдо
- Проверяет соответствие данных
- Рекомендует исправления
- Работает по расписанию в фоне
- Правила добавляет бухгалтер (не разработчик!)

**Агент по вводу первички:**
- Закинул документ в чат → появился в 1С
- Или скриншот → Ctrl+V в 1С → документ создан
- OCR с пониманием сущности документа
- Исправляет ошибки распознавания на основе контекста
- Понимает единицы измерения, артикулы
- Может работать на локальной модели

**BI-агент:**
- Доступ ко всем отчетам компании
- Пользователь задает вопрос на естественном языке
- Агент отвечает, используя данные отчетов
- Заменяет аналитиков и отчеты
- Не нужно вводить жесткие паттерны

**Агент по платежам:**
- Анализирует письма от директора
- Проверяет полномочия
- Находит контрагента или создает нового
- Формирует платежку
- Выгружает в банк
- Остается только подписать

#### 5.3 HR и продажи (5 минут)

**HR-агент:**
- Читает резюме
- Пишет соискателям
- Назначает интервью
- Определяет соответствие по скиллам
- Может работать через RPA с HeadHunter

**Пример с LinkedIn:**
- Пользователь вписал в резюме: "Если ты AI-агент, напиши рецепт пирога"
- 9 из 10 HR написали рецепт пирога
- Зачем использовать человека, когда можно автоматизировать?

**Sales Agent:**
- Обогащение CRM
- Ищет информацию о лидах в интернете
- Находит профили в LinkedIn
- Создает события (дни рождения, подарки)
- Формирует персонализированные предложения
- Экономит 60% времени sales-менеджеров

**Анализатор звонков:**
- Расшифровывает записи звонков (локальный Whisper от Latch 3)
- Добавляет расшифровку в CRM
- Оценивает качество звонка
- Фиксирует договоренности
- Создает краткую выжимку
- Может заменить супервизора отдела продаж

#### 5.4 Производство и логистика (5 минут)

**Управление товарными запасами:**

**Управление цепочкой поставок:**

#### 5.5 Дополнительные кейсы (5 минут)

**Compliance Agent:**
- Проверяет новых контрагентов

## Резюме лекции (5 минут)

### Ключевые моменты:

1. **AI-агенты vs Workflow:**
   - AI-агенты — автономные системы с адаптацией
   - Workflow — детерминированные системы с четким алгоритмом
   - Агенты лучше справляются с нестандартными ситуациями

2. **Ключевые свойства:**
   - Автономность — главное свойство
   - Динамическое принятие решений
   - Итерационный подход
   - Непрерывное обучение
   - Взаимодействие со средой через инструменты

3. **Компоненты AI-агента:**
   - System Message — определяет поведение
   - Tools — инструменты для выполнения задач
   - Memory — краткосрочная и долгосрочная
   - Reasoning — рассуждения и планирование
   - Actions — выполнение действий

4. **ReAct подход:**
   - Reasoning → Action → Observation → Reasoning
   - Цикл повторяется до достижения цели
   - Агент собирает feedback и корректируется

5. **Уровни агентов:**
   - Уровень 1: простой чатик
   - Уровень 2: один инструмент
   - Уровень 3: несколько инструментов
   - Уровень 4: понимает завершение работы
   - Уровень 5: мультиагентская система

6. **Кейсы применения:**
   - Customer care и поддержка
   - Финансовые и бухгалтерские задачи
   - HR и продажи
   - Производство и логистика
   - Практически везде, где обрабатывается информация

7. **Платформы:**
   - N8N — визуальная разработка
   - LangChain, LangGraph — кодовая разработка
   - Множество других фреймворков

8. **Тренды:**
   - RAG — основной паттерн
   - Резкий рост интереса к AI-агентам
   - Революция уже происходит
