### Блок 1: Введение в N8N (10 минут)

#### 1.1 Что такое N8N (5 минут)

**N8N** — это платформа для разработки автоматизации рабочих процессов.

**Ключевые характеристики:**
- **Low-code платформа** — здесь мы не столько пишем код, сколько работаем с помощью добавления нод (узлов)
- Огромное количество нод под разные случаи жизни и под разные интеграции
- Визуальное программирование через интерфейс

**Основная цель N8N:**
- Автоматизация рабочих процессов
- Создание цепочек из нод для решения задач
- Интеграция различных сервисов и инструментов

---

#### 1.2 Ключевые понятия (5 минут)

**Workflow (Рабочий процесс):**
- Весь процесс состоит из узлов
- Workflow — это набор узлов, соединенных между собой
- Можно сохранить как JSON-файл

**Credentials (Креденшнлы):**
- Доступы к разным инструментам по API
- Примеры: PostgreSQL, Qdrant, OpenAI, Google Таблицы
- Настройка один раз, использование в разных workflow

**Узлы (Nodes):**
- Каждый элемент в workflow — это узел
- Различные типы узлов для разных задач

**Триггеры (Triggers):**
- Узлы, которыми процесс запускается
- Может быть только один триггер в workflow
- Примеры: Chat Trigger, Webhook Trigger, Manual Trigger

**Action Nodes:**
- Все узлы, которые идут после триггера
- Выполняют действия в рабочем процессе

**Execution (Выполнение):**
- Каждый запуск рабочего процесса называется execution
- Можно отслеживать в разделе Executions
- Позволяет видеть все выводы и отлаживать процесс

### Блок 2: Развертывание N8N (15 минут)

#### 2.1 Варианты развертывания (5 минут)

**N8N — open-source проект:**
- Можно развернуть практически как угодно
- Можно у себя прямо на компе поднять, даже без Docker
- Можно развернуть в Docker-контейнере на любом удаленном сервере
- Есть N8N Cloud — облачное решение от разработчиков

**Преимущества N8N Cloud:**
- Есть агент, который помогает разобраться
- Агент смотрит, что вы делаете, и может дать рекомендации
- Есть пробный период (14 дней)
- Официальная подписка (около 20-24 долларов в месяц)

---

#### 2.2 Самостоятельное развертывание (5 минут)

**Плюсы самостоятельного развертывания:**
- Полный контроль над системой
- Конфиденциальность данных
- Бесплатно (кроме стоимости сервера)

**Минусы самостоятельного развертывания:**
- Сложнее настройка доступа к сервисам (например, Google через OAuth)
- Нужно получать SSL-сертификаты для веб-хуков
- Нужен домен для работы веб-хуков
- Нет встроенного ассистента
- Сложнее обновления

**Особенности:**
- Много нод работают через веб-хуки (Telegram и др.)
- Для веб-хуков нужен домен и SSL-сертификат
- Самостоятельное развертывание чуть сложнее

---

#### 2.3 Рекомендации по выбору (5 минут)

**Для тестового стенда:**
- Можно разворачивать у себя локально
- Для круглосуточной работы нужен сервер

**Для продакшена:**
- Должно быть на сервере
- Либо в облаке (если нет конфиденциальной информации)
- Либо на собственном сервере

**Рекомендация:**
- Если есть лишние 24 доллара в месяц, то освоение N8N в облаке будет сильно проще
- Для новичков облачное решение упрощает порог вхождения

### Блок 3: Создание базового ИИ-агента (20 минут)

#### 3.1 Настройка базового агента (10 минут)

**Компоненты базового ИИ-агента:**

1. **Chat Trigger** — триггер для сообщений внутри интерфейса N8N
   - Запускает процесс при получении сообщения в чате

2. **AI Agent** — узел для работы с ИИ
   - Подключается к языковой модели

3. **Chat Model** — выбор языковой модели
   - Пример: GigaChat 2 Max (бесплатная модель)
   - Можно использовать различные модели через credentials

**Настройка промпта:**
- **Prompt** — то, что отправляется боту
- **System Message** — системное сообщение, которое добавляется в каждый чат
  - Определяет поведение агента
  - Пример: "Ты эксперт по кинематографу 90-х годов"

**Память (Memory):**
- Без памяти каждый чат изолированный
- Агент не помнит предыдущие сообщения
- **Simple Memory** — простая память, которая хранит последние N сообщений
- Настройка: количество предыдущих сообщений для хранения (например, 5)

**Как работает память:**
- Берет всю историю сообщений
- Отсылает в модель: системный промпт, запрос от human, ответ агента, следующий вопрос
- На основании этого агент может понять контекст предыдущих сообщений

---

#### 3.2 Демонстрация работы агента (10 минут)

**Пример использования:**
- Создание агента с System Message "Ты эксперт по кинематографу 90-х годов"
- Без памяти: агент не помнит предыдущие сообщения
- С памятью: агент помнит контекст и может отвечать на вопросы о предыдущих сообщениях

**Интеграция с Telegram:**
- Можно настроить чат-бота для Telegram
- Использование Telegram триггера вместо Chat Trigger
- Для работы нужен домен и SSL-сертификат (при самостоятельном развертывании)

**Интеграция с другими мессенджерами:**
- Можно настроить чат-бота для Max Messenger
- Большое комьюнити, много примеров интеграций

### Блок 4: Работа с данными и JSON (10 минут)

#### 4.1 Workflow как JSON (5 минут)

**Концепция:**
- Все, что мы рисуем в N8N, представлено в виде JSON
- Можно сохранить workflow как JSON-файл
- Можно скопировать workflow (Ctrl+C) и вставить (Ctrl+V)
- Можно программировать workflow с помощью нейросетей, так как нейросети хорошо генерируют JSON

**Структура:**
- Workflow — это JSON-код
- Каждая нода представлена в JSON
- Можно редактировать напрямую в JSON (но не рекомендуется для новичков)

---

#### 4.2 Типы данных и Expressions (5 минут)

**Типы данных в N8N:**
- **String** (строка) — текст
- **Number** (число) — числа
- **Boolean** (булево) — true/false
- **Array** (массив) — массив данных
- **Object** (объект) — объект
- **Binary** (бинарные данные) — файлы, изображения и т.д.

**Expressions (Выражения):**
- Динамические выражения для работы с данными
- Можно перетягивать значения из предыдущих нод
- При редактировании можно увидеть, как будет выглядеть значение после выполнения выражения
- Позволяют использовать данные из предыдущих нод в текущей ноде

**Пример использования:**
- В System Message можно вставить expression из предыдущей ноды
- Значение будет динамически подставляться при выполнении

### Блок 5: Основные узлы N8N (40 минут)

#### 5.1 Manual Trigger (2 минуты)

**Назначение:**
- Самый простой триггер
- Используется для отладки
- Эмулирует запуск рабочего процесса вручную

**Применение:**
- Тестирование workflow
- Отладка процессов

---

#### 5.2 Узел IF (5 минут)

**Назначение:**
- Выполняет условия в зависимости от истинности или ложности
- Два выхода: True и False

**Настройка:**
- Выбор поля для сравнения
- Выбор условия: равно, не равно, содержит и т.д.
- Конвертация типов (если требуется)
- Опция "ignore case" — игнорирование регистра

**Пример:**
- Проверка равенства кода единице
- Если код равен 1 — выполняется ветка True
- Если нет — выполняется ветка False

**Особенности:**
- Можно сравнивать строки и числа
- Важно правильно указать тип данных для сравнения

---

#### 5.3 Узел SWITCH (5 минут)

**Назначение:**
- Направление по разным путям в зависимости от условия
- Более продвинутый вариант IF

**Настройка:**
- Можно задать множество routing rules (правил маршрутизации)
- Каждое правило ведет в отдельную ветку
- Можно переименовать выходы для удобства

**Пример:**
- Если код равен 1 — первая ветка
- Если код равен 2 — вторая ветка
- Все остальное — default ветка

**Особенности:**
- Можно написать множество условий
- Можно задать множество путей

---

#### 5.4 Узел Filter (3 минуты)

**Назначение:**
- Фильтрует данные, отсеивая ненужные
- Либо пропускает, либо не пропускает данные

**Принцип работы:**
- Если условие выполняется — данные проходят дальше
- Если условие не выполняется — данные не проходят дальше

**Пример:**
- Если код равен 1 — пропускает дальше
- Если нет — не пропускает

---

#### 5.5 Узел Merge (7 минут)

**Назначение:**
- Объединяет данные из разных веток рабочего процесса

**Режимы работы:**

1. **Append (Дополнить)** — безусловно просто соединяет два массива
   - Union all условно
   - Просто объединяет все элементы

2. **Keep Matches** — объединяет по условию, оставляя только пересекающиеся
   - Ищет совпадения по указанному полю
   - Оставляет только те элементы, которые есть в обеих таблицах
   - Данные берутся из второй таблицы

3. **Keep Non Matches** — оставляет только не пересекающиеся
   - Выдает значения, которые есть только в одной таблице

4. **Keep Everything** — сохраняет все, но без дубликатов
   - Left join и right join реализованы
   - Можно дополнить данные из одной таблицы данными из другой

**Особенности:**
- Можно подключить множество источников (не только два)
- В настройках можно указать несколько источников

---

#### 5.6 Узел Loop (Split In Batches) (7 минут)

**Назначение:**
- Разбивает массив на элементы и обрабатывает каждый элемент отдельно
- Цикл по элементам

**Настройка:**
- Тип: Everything (обрабатывает все элементы)
- На вход подается массив
- На выходе каждый элемент обрабатывается отдельно

**Применение:**
- Обработка каждого элемента массива отдельно
- Преобразование данных каждого элемента
- После обработки все элементы собираются обратно в массив

**Пример:**
- На входе таблица с кандидатами
- Каждый кандидат обрабатывается отдельно
- Данные преобразуются (переименование полей)
- На выходе обработанный массив

---

#### 5.7 Узел Set (Edit Fields) (5 минут)

**Назначение:**
- Преобразование данных
- Изменение структуры данных
- Переименование полей

**Применение:**
- Переименование колонок (например, "роль" → "должность")
- Выбор нужных полей из данных
- Изменение значений полей

**Пример:**
- На входе: массив с полями "кандидат" и "роль"
- На выходе: массив с полями "сотрудник" и "должность"
- Значения остаются те же, меняются только названия полей

---

#### 5.8 Узел Aggregate (3 минуты)

**Назначение:**
- Комбинация и объединение данных
- Преобразование данных специальным образом

**Применение:**
- После цикла (Loop) элементы обрабатываются по отдельности
- Aggregate собирает все элементы обратно в единый массив
- Преобразует разрозненные элементы в единую таблицу

**Пример:**
- После Loop получили 5 элементов, каждый выполнился отдельно
- Aggregate собирает их в единый JSON-массив

---

#### 5.9 Узел Sort (5 минут)

**Назначение:**
- Сортировка данных

**Настройки:**
- Выбор поля для сортировки
- Тип сортировки: по возрастанию или по убыванию
- Опция random — случайная сортировка
- Можно использовать код для сложной сортировки

**Пример:**
- Сортировка по должности по убыванию
- Эксперт будет на первом месте, дальше по алфавиту

---

#### 5.10 Узел Split Out (5 минут)

**Назначение:**
- Разделение данных
- Отделение определенных полей

**Режимы:**
- **Select Other Fields** — выбрать поля для вывода
- **All Other Fields** — вывести все поля
- Убирает лишние столбцы, оставляет только нужные

**Применение:**
- Если нужно обрабатывать данные пошагово
- Разделение массива на отдельные поля
- Отделение определенных полей от остальных

**Пример:**
- На входе таблица
- На выходе отдельные поля (например, "сотрудник" и "должность")

---

#### 5.11 Узел Code (3 минуты)

**Назначение:**
- Написание кода на JavaScript или Python
- Для случаев, когда не хватает функционала N8N

**Применение:**
- Для опытных разработчиков
- Можно использовать AI-кодинг (vibe-кодинг) с помощью нейросетей

**Особенности:**
- С отладкой тут все сложно
- Код либо выполняется, либо не выполняется
- Отлаживать сложно

---

#### 5.12 Узел Execute Workflow (5 минут)

**Назначение:**
- Запуск другого рабочего процесса из текущего

**Применение:**
- Разделение сложных процессов на модули
- Создание специализированных workflow
- Вызов одного workflow из другого

**Настройка:**
- Выбор workflow для запуска
- Передача данных в другой workflow
- Получение результата обратно

**Пример:**
- Основной workflow обрабатывает данные
- Вызывает специализированный workflow для обработки каждого элемента
- Получает результат обратно

---

#### 5.13 Узел Wait (3 минуты)

**Назначение:**
- Ожидание определенного времени перед продолжением выполнения

**Настройки:**
- Задержка в секундах, минутах, часах, днях
- До определенного времени
- В специальную дату/время

**Применение:**
- Задержки между запросами к API
- Ожидание определенного времени
- Сценарии согласования (например, дождаться ответа генерального директора)

---

#### 5.14 Узел Do Nothing (1 минута)

**Назначение:**
- Абсолютно бессмысленный узел
- Не имеет параметров и ничего не делает

**Применение:**
- Эстетическое обозначение завершения рабочего процесса
- Визуальное удовольствие

---

### Блок 6: HTTP Request и WebHook (10 минут)

#### 6.1 HTTP Request (5 минут)

**Назначение:**
- Подключение к сервисам через API
- Вызов внешних API

**Применение:**
- Если нужного сервиса нет в списке интеграций N8N
- Можно настроить HTTP-запрос вручную для любого сервиса с API
- Настройка URL, авторизации (API ключ), данных запроса

**Пример:**
- Вызов API 1С через веб-сервис
- Настройка Apache для поднятия веб-сервиса в 1С
- Вызов через HTTP Request из N8N

---

#### 6.2 WebHook (5 минут)

**Назначение:**
- Триггеры для получения данных из внешних источников
- Запуск рабочих процессов по событиям в режиме реального времени

**Применение:**
- Слушаем, чтобы к нам что-то обратилось
- Реакция на внешние события
- Вызов внешних сервисов

**Примеры:**
- WebHook для Telegram (на прием сообщения)
- WebHook для получения данных от 1С
- 1С может вызывать N8N через веб-хуки

**Особенности:**
- JSON-данные замечательно воспринимаются LLM
- Можно передавать данные между системами в формате JSON

---

### Блок 7: Обработка ошибок (5 минут)

#### 7.1 Механизм обработки ошибок

**Назначение:**
- Настройка выполнения других рабочих процессов при возникновении сбоев
- Мониторинг ошибок в процедурах

**Настройка:**
- В Settings workflow выбрать Error Workflow
- Указать рабочий процесс, который запускается при ошибке
- Подключить ноду обработки ошибок к любой ноде

**Компоненты:**
- **Error Workflow** — workflow для обработки ошибок
- Можно настроить оповещение (в Telegram, на почту и т.д.)
- Можно настроить обработку ошибки

**Применение:**
- Для продакшена важно мониторить ошибки
- Моментальное уведомление об ошибках
- Отправка сообщений команде для исправления

## Резюме лекции (5 минут)

### Ключевые моменты:

1. **N8N** — low-code платформа для автоматизации рабочих процессов
   - Программирование через интерфейс (добавление нод)
   - Workflow записывается как JSON-код
   - Огромное количество нод под разные случаи жизни

2. **Ключевые понятия:**
   - Workflow — рабочий процесс из узлов
   - Credentials — доступы к инструментам по API
   - Узлы (Nodes) — элементы workflow
   - Триггеры — запускают процесс (может быть только один)
   - Execution — выполнение процесса

3. **Развертывание:**
   - Самостоятельно на сервере (сложнее, но дешевле)
   - N8N Cloud (удобнее, но платно, около 24 долларов в месяц)
   - Для новичков рекомендуется облачное решение

4. **ИИ-агенты:**
   - Базовый агент: Chat Trigger + AI Agent + Chat Model
   - System Message определяет поведение
   - Память (Simple Memory) необходима для контекста
   - Без памяти каждый чат изолированный

5. **Обработка данных:**
   - Expressions — динамические выражения для работы с данными
   - Типы данных: string, number, boolean, array, object, binary
   - JSON как формат хранения workflow
   - Можно программировать workflow с помощью нейросетей

6. **Основные узлы:**
   - IF, SWITCH — условия
   - Filter — фильтрация
   - Merge — объединение данных (Append, Keep Matches, Keep Non Matches, Keep Everything)
   - Loop (Split In Batches) — обработка по элементам
   - Set (Edit Fields) — преобразование данных
   - Aggregate — объединение данных после цикла
   - Sort — сортировка
   - Split Out — разделение данных
   - Code — написание кода
   - Execute Workflow — запуск другого workflow
   - Wait — ожидание
   - Do Nothing — завершение процесса

7. **HTTP Request и WebHook:**
   - HTTP Request — подключение к любым API
   - WebHook — триггеры из внешних сервисов
   - Интеграция с 1С через веб-сервисы

8. **Обработка ошибок:**
   - Настройка Error Workflow
   - Моментальное уведомление об ошибках

9. **Память:**
   - Simple Memory — простая память контекста
   - Векторная база данных — для более продвинутых случаев

10. **Обучение:**
    - Документация и сообщество
    - Практика создания собственных проектов
    - Воспроизведение примеров руками
