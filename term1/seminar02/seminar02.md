# Семинар 2: Создание конфигурации 1С:Предприятие с нуля — справочники, документы, регистры

**Продолжительность:** ~2 часа  
**Целевая аудитория:** Студенты 2 курса, Факультет информационных технологий

## Цель семинара
Создать с нуля конфигурацию «Система учёта предприятия»: настроить справочники, перечисления, документы, регистры накопления, конструктор движений, формы и программный код для автоматизации заполнения реквизитов.

- Интерфейс конфигуратора 1С:Предприятие

### Блок 1: Настройка конфигурации и создание справочников (~40 минут)

#### 1.1 Свойства конфигурации (5 минут)

**Создание новой информационной базы:**
1. Создать новую пустую ИБ, запустить конфигуратор
2. Открыть конфигурацию → двойной щелчок или правая кнопка → Свойства

**Заполнение свойств:**
- **Имя** — без пробелов, начинается с буквы: `СистемаУчета`
- **Синоним** — «Система учёта предприятия, редакция 1.0»
- **Поставщик** — имя разработчика
- **Версия** — `1.0.0.1`
  - Формат: `редакция.подредакция.сборка` (аналогично типовым: УХ 3.1.14.26)
- **Краткая/подробная информация** — описание конфигурации

**Сценарий создаваемой ИБ:**
- Учёт номенклатуры (товары и услуги)
- Учёт контрагентов и договоров (взаиморасчеты в разрезе договоров)
- Складской учёт
- Документы: поступление и реализация товаров
- Возможность печати документов
- Параметризация учёта по складам и ценообразованию

#### 2.3 Конструктор движений документов (10 минут)

**Для ПоступлениеТоваров** (вкладка «Движения»):

Настройки:
- Движение разрешено
- Оперативное проведение разрешено
- Движения удаляются при отмене проведения автоматически

Движения по **Взаиморасчетам**:
- Тип: **Приход** (мы должны поставщику)
- Контрагент → `Контрагент` документа
- Договор → `Договор` документа
- Сумма → `Сумма` документа

Движения по **ОстаткиТоваров**:
- Тип: **Приход**
- Источник: **табличная часть Товары** (для каждой строки — своё движение)
- Номенклатура → `Товары.Номенклатура`
- Количество → `Товары.Количество`
- Склад → `Склад` документа (из шапки)

**Разница в подходе:**
- Взаиморасчеты — одна запись на весь документ (берётся `Сумма` из шапки)
- Остатки — цикл по табличной части, для каждой строки создаётся движение

**Для РеализацияТоваров** — аналогично, но тип движений: **Расход**:
- По взаиморасчетам — Расход (покупатель становится должен нам)
- По остаткам — Расход (товар уходит со склада)

### Блок 3: Формы и программный код (~50 минут)

#### 3.1 Автозаполнение дат договора при создании (10 минут)

**Проблема:** При открытии существующего договора даты перезаписываются.

**Решение:** Обработчик «При создании на сервере» (а не «При открытии» на клиенте — нам нужен доступ к данным базы).

**Проверка нового объекта:**

В отладчике: у нового объекта `Объект.Ссылка` — не заполнена (пустая ссылка), у существующего — заполнена.

```1c
// Обработчик "При создании на сервере" в модуле формы элемента
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если Не ЗначениеЗаполнено(Объект.Ссылка) Тогда
        Объект.НачалоДоговора = НачалоГода(ТекущаяДата());
        Объект.КонецДоговора  = КонецГода(ТекущаяДата());
    КонецЕсли;
КонецПроцедуры
```

**Ключевые функции:**
- `ТекущаяДата()` — возвращает текущую системную дату (доступна везде: клиент, сервер)
- `НачалоГода(Дата)` — первый момент года для указанной даты
- `КонецГода(Дата)` — последний момент года для указанной даты
- `ЗначениеЗаполнено(Значение)` — возвращает Истина, если значение не пустое

**Полная форма условной конструкции:**

```1c
Если <условие1> Тогда
    // код
ИначеЕсли <условие2> Тогда
    // другой код
Иначе
    // иначе
КонецЕсли;
```

#### 3.2 Форма выбора договоров с фильтрацией (10 минут)

**Проблема:** При выборе договора в документе показываются все договоры всех контрагентов (и помеченные на удаление).

**Шаг 1 — Связь параметров выбора реквизита «Договор» в документе:**

В свойствах реквизита `Договор` (в документе ПоступлениеТоваров) → «Связи параметров выбора»:
- Параметр: `Владелец`, значение: `Контрагент` (реквизит документа)

**Результат:** При выборе договора показываются только договоры выбранного контрагента.

**Шаг 2 — Форма выбора справочника ДоговорыКонтрагентов:**

Добавить форму выбора (отдельная форма). В конструкторе выбрать поля:
- Код, Наименование, НачалоДоговора, КонецДоговора

**Динамический список с отбором:**
- Установить «Произвольный запрос»
- Открыть настройки → вкладка «Настройки» → Отбор:
  - `ПометкаУдаления` = `Ложь`

```
// Отбор не показывает помеченные на удаление договора
ПометкаУдаления = Ложь
```

**Важно:** Те же отборы пользователь может настроить вручную через меню «Ещё → Настроить список».

#### 3.3 Автоподбор договора при изменении контрагента (15 минут)

**Задача:** При смене контрагента в документе автоматически подставлять актуальный договор.

**Архитектура решения:**

```
Форма документа (клиент)
    → ПриИзмененииКонтрагента()  [клиент]
        → ЗаполнитьДоговорНаСервере()  [сервер]
            → Справочники.ДоговорыКонтрагентов.ПолучитьДоговор()  [модуль менеджера]
```

**Создание обработчика:** В свойствах поля «Контрагент» на форме → событие «При изменении» → «Создать на клиенте и процедуру на сервере».

**Функция ПолучитьДоговор в модуле менеджера ДоговорыКонтрагентов:**

```1c
// Модуль менеджера справочника ДоговорыКонтрагентов
Функция ПолучитьДоговор(Контрагент, Дата) Экспорт
    ЗапросДоговоров = Новый Запрос;
    ЗапросДоговоров.Текст =
        "ВЫБРАТЬ
        |    ДоговорыКонтрагентов.Ссылка КАК Договор
        |ИЗ
        |    Справочник.ДоговорыКонтрагентов КАК ДоговорыКонтрагентов
        |ГДЕ
        |    НЕ ДоговорыКонтрагентов.ПометкаУдаления
        |    И ДоговорыКонтрагентов.Владелец = &Контрагент
        |    И ДоговорыКонтрагентов.НачалоДоговора <= &Дата
        |    И ДоговорыКонтрагентов.КонецДоговора >= &Дата
        |УПОРЯДОЧИТЬ ПО
        |    ДоговорыКонтрагентов.НачалоДоговора УБЫВ";

    ЗапросДоговоров.УстановитьПараметр("Контрагент", Контрагент);
    ЗапросДоговоров.УстановитьПараметр("Дата", Дата);

    Выборка = ЗапросДоговоров.Выполнить().Выбрать();
    Если Выборка.Следующий() Тогда
        Возврат Выборка.Договор;
    Иначе
        Возврат Неопределено;
    КонецЕсли;
КонецФункции
```

**Вызов из формы документа (серверная процедура):**

```1c
&НаСервере
Процедура ЗаполнитьДоговорНаСервере()
    Объект.Договор = Справочники.ДоговорыКонтрагентов.ПолучитьДоговор(
        Объект.Контрагент, Объект.Дата);
КонецПроцедуры
```

**Ключевые понятия:**

- **Запрос** — программный объект для получения данных из ИБ
  - `Новый Запрос` — создание объекта
  - `.Текст` — текст запроса на языке запросов 1С
  - `.УстановитьПараметр("Имя", Значение)` — передача параметров в запрос
  - `.Выполнить()` — выполнение и получение результата
  - `.Выбрать()` — создание выборки для обхода результата
  - `Выборка.Следующий()` — переход к следующей записи, возвращает Булево

- **Конструктор запроса** — правая кнопка в модуле → «Конструктор запроса»

- **Модуль менеджера** — содержит методы, не привязанные к конкретному объекту (аналог статических методов в ООП). Доступен через `Справочники.ИмяСправочника.МетодМенеджера()`

- **Модуль объекта** — содержит методы, работающие с конкретным экземпляром объекта

- **Экспорт** — ключевое слово, делающее функцию/процедуру доступной из других модулей

- **Процедура vs Функция** — функция обязательно возвращает значение через `Возврат`; процедура — только выполняет действия

- **Неопределено** — аналог `null` в других языках

**Упорядочивание результата:** `УПОРЯДОЧИТЬ ПО НачалоДоговора УБЫВ` — первым будет самый свежий договор.

#### 3.4 Автозаполнение цены и пересчёт суммы (15 минут)

**Задача:** При выборе номенклатуры — подставлять цену; при изменении цены или количества — пересчитывать сумму.

**Общий модуль «Ценообразование»:**

Добавить новый общий модуль (правая кнопка на «Общие модули»):
- Имя: `Ценообразование`
- Признак: **Сервер** (код выполняется на сервере)
- Не глобальный (вызов через `Ценообразование.МетодМодуля()`)

```1c
// Общий модуль "Ценообразование"
Функция ПолучитьЦенуНоменклатуры(Номенклатура) Экспорт
    Запрос = Новый Запрос;
    Запрос.Текст =
        "ВЫБРАТЬ
        |    Номенклатура.Цена КАК Цена
        |ИЗ
        |    Справочник.Номенклатура КАК Номенклатура
        |ГДЕ
        |    Номенклатура.Ссылка = &Номенклатура";
    Запрос.УстановитьПараметр("Номенклатура", Номенклатура);
    Выборка = Запрос.Выполнить().Выбрать();
    Если Выборка.Следующий() Тогда
        Возврат Выборка.Цена;
    Иначе
        Возврат 0;
    КонецЕсли;
КонецФункции
```

**Серверная функция в форме документа:**

```1c
&НаСервере
Функция ПолучитьЦенуНаСервере(Номенклатура)
    Возврат Ценообразование.ПолучитьЦенуНоменклатуры(Номенклатура);
КонецФункции
```

**Клиентский обработчик при изменении номенклатуры:**

```1c
&НаКлиенте
```

**Пересчёт суммы (клиент):**

```1c
&НаКлиенте
Процедура ПересчитатьСумму()
    Строка = Элементы.Товары.ТекущиеДанные;
    Строка.Сумма = Строка.Количество * Строка.Цена;
КонецПроцедуры
```

Назначить `ПересчитатьСумму()` обработчиком «При изменении» для полей **Количество** и **Цена** в табличной части.

**Подсчёт суммы документа перед записью:**


**Вывод движений документа на форму:**

В конструкторе формы документа → «Командный интерфейс» → вкладка «Перейти» → включить видимость команд «Взаиморасчеты» и «ОстаткиТоваров».

**Разграничение клиент/сервер:**

| Действие | Где выполнять |
|---|---|
| Обращение к ИБ, запросы | Сервер |
| Тяжёлые вычисления | Сервер |
| Простая арифметика (цена × количество) | Клиент (лишнее переключение на сервер дороже) |
| Работа с текущей строкой табличной части | Клиент |

> **Важно:** Переключение контекста клиент↔сервер — дорогостоящая операция. Минимизируйте количество таких переключений, группируя серверные вызовы.
