# Quiz: Лекция 1 — Введение в Библиотеку стандартных подсистем (БСП)

## Вопрос 1
**Что такое Библиотека стандартных подсистем (БСП)?**

1. Готовая учётная система для бухгалтерии
2. Набор универсальных функциональных подсистем — инструментарий для разработки решений (верно)
3. Набор отраслевых решений для ERP
4. Конфигурация для управления холдингом

2. Базовая функциональность, Обновление версии, Пользователи (верно)

1. Для развёртывания рабочей базы предприятия

## Вопрос 4
**В чём заключается «правило фасада» при работе с БСП?**

1. Все процедуры БСП нужно вызывать напрямую без обёрток
2. Использовать только публичный API; не вызывать служебные модули и процедуры (верно)
3. Создавать фасадные модули для каждой подсистемы
4. Скрывать все вызовы БСП внутри своих общих модулей

---

## Вопрос 5
**Какие три легальных способа интеграции своих объектов с БСП рассмотрены в лекции?**

1. Копирование кода, наследование, делегирование
2. Определяемые типы, переопределяемые модули, интеграция через метаданные (контракт данных) (верно)
3. Подписки на события, регламентные задания, HTTP-сервисы
4. Внешние обработки, расширения конфигурации, внешние компоненты

---

## Вопрос 6
**Что произойдёт, если разработчик вызовет служебную процедуру БСП напрямую?**

1. Всё будет работать корректно, так как процедура экспортная
2. Фирма 1С гарантирует обратную совместимость служебного кода
3. При обновлении БСП процедура может быть изменена или удалена, что приведёт к ошибке (верно)
4. Система автоматически перенаправит вызов в публичный модуль

---

## Вопрос 7
**В чём разница между самостоятельными и интегрируемыми подсистемами БСП?**

1. Самостоятельные платные, интегрируемые — бесплатные
2. Самостоятельные не требуют доработок объектов, интегрируемые требуют внесения изменений в объекты (верно)
3. Самостоятельные используются в РФ, интегрируемые — в международных решениях
4. Разницы нет, это синонимы

---

## Вопрос 8
**Как работает интеграция через определяемые типы?**

1. Подсистема создаёт новые объекты метаданных автоматически
2. Объект добавляется в состав определяемого типа, и подсистема начинает его обрабатывать (верно)
3. Разработчик создаёт копию определяемого типа для каждого своего объекта
4. Определяемые типы используются только для справочников

## Вопрос 10
**Почему знание БСП важно для разработчика 1С с точки зрения карьеры?**

1. БСП обязательна для сертификации 1С
2. Работодатели требуют знание БСП в вакансиях; незнание ведёт к потере времени на реализацию уже готового функционала (верно)
3. Без БСП невозможно создавать конфигурации
4. БСП повышает производительность платформы
