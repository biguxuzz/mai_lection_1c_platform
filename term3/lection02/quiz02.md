# Quiz: Лекция 2 — Базовая функциональность БСП (часть 1)


## Вопрос 4
**Для чего в базовой функциональности БСП нужен механизм (справочник) «Идентификаторы объектов метаданных и объектов расширений»?**

1. Чтобы заменить собой дерево метаданных в конфигураторе
2. Чтобы хранить только объекты расширений, без объектов основной конфигурации
3. Чтобы вручную заводить коды для каждого элемента справочника
4. Чтобы дублировать в ИБ сведения об объектах и использовать их в сценариях (в т.ч. запросах) без постоянных обращений к структуре метаданных «в лоб» (верно)

---

## Вопрос 5
**Почему новый документ, добавленный в конфигураторе, может ещё не отображаться в справочнике идентификаторов объектов метаданных?**

1. Справочник заполняется только при покупке лицензии ИТС
2. Идентификаторы создаются только для объектов с префиксом «Демо»
3. Актуализация справочника выполняется при обновлении информационной базы, а не сразу при сохранении метаданных (верно)
4. Документы в БСП не получают идентификаторы

---

## Вопрос 6
**Какими способами можно инициировать обновление информационной базы с точки зрения параметров запуска (по лекции)?**

1. Параметры запуска из конфигуратора (в т.ч. дополнительные), командная строка, аналогичные параметры в среде разработки (верно)
2. Только интерактивно через «Администрирование» в режиме предприятия
3. Только через обработку «Первое внедрение БСП»
4. Только при полном отключении поддержки конфигурации

---

## Вопрос 7
**Чем в лекции отличали «критичное» обновление от переносимого в фон?**

1. Критичное выполняется только в файловой базе, фоновое — только на сервере
2. Критичное — это любое переименование, фоновое — любое добавление реквизита
3. Критичное — без завершения которого система не имеет смысла (пользователь ждёт); некритичное можно отложить/выполнить фоном (верно)
4. Различий нет, это синонимы

---

## Вопрос 8
**Зачем в демобазе БСП нужен режим технического специалиста?**

1. Чтобы отключить все права пользователей
2. Чтобы ускорить проведение документов
3. Только для просмотра журнала регистрации
4. Чтобы получить доступ к служебным спискам и функциям (в т.ч. идентификаторам метаданных), не выведенным в основной интерфейс (верно)

## Вопрос 9
**Что рекомендовано сделать в dev-контуре после переименования объекта метаданных, чтобы синхронизировать вспомогательные данные (идентификаторы)?**

1. Удалить информационную базу и создать заново
2. Применить изменения конфигурации, перезапустить сеанс при необходимости и выполнить обновление вспомогательных данных (обработка из «Инструментов разработчика») (верно)
3. Достаточно изменить синоним — справочник обновится сам при следующем входе пользователя
4. Переименование объектов метаданных в БСП запрещено
