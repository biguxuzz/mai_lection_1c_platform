**Продолжительность:** ~55 минут

## Цель лекции

Разобрать подсистему **«Длительные операции»** БСП: почему **нельзя** выполнять долгую серверную работу **синхронно** из формы; куда выносить код (**модуль менеджера** документа); каким API (**`ДлительныеОперации`**, **`ДлительныеОперацииКлиент`**) запускать фоновое выполнение, показывать **ожидание** и по **оповещению** забирать результат из **временного хранилища**. Зафиксировать типовые **антипаттерны**: **`Выполнить().Выгрузить()`** на больших данных, **активное ожидание** в цикле, вызов **`Пауза`** не из того контекста.

## Необходимые предзнания

- Лекции 1–2 и семинары по БСП: базовая функциональность, клиент/сервер, **временное хранилище** (понятие)
- Управляемые формы, команды формы, `&НаСервере`, **экспортные** процедуры для оповещений
- Документ **«Реализация товаров и услуг»** с табличной частью **«Товары»** (учебная демоконфигурация)

## План лекции (~55 минут)

### Блок 1: Постановка демо и типовые ошибки размещения кода (8 минут)

#### 1.1 Сценарий

- Документ **«Реализация товаров и услуг»** (подсистема курса в дереве конфигурации).
- Команда формы **«Заполнение табличной части»** — имитация **длительной** операции (порядка **8–30 секунд** в учебных целях).

#### 1.2 Почему не в модуле формы

**Правило:** логику массового заполнения данных документа не размещают в **модуле формы**, если её могут понадобиться **вызвать программно** (без открытия формы).

**Куда выносить:** **`модуль менеджера`** этого документа — экспортная функция/процедура, строка вида **`Документы.РеализацияТоваровУслуг.ИмяМетода`** используется и из формы, и из другого серверного кода.

#### 1.3 Наивный «длинный» вызов с клиента

Прямой **синхронный** вызов серверной функции из обработчика команды приводит к **блокировке интерфейса** на всё время выполнения: пользователь не может работать с приложением.

### Блок 2: Запрос, память и имитация задержки (10 минут)

#### 2.1 Антипаттерн: `Запрос.Выполнить().Выгрузить()`

Для **тяжёлых** систем результат **`Выгрузить()`** может **съесть память** сервера (вся выборка сразу в оперативной памяти).

**Лучше по смыслу нагрузки:** построчная/пакетная обработка результата без полной выгрузки в таблицу значений — в зависимости от задачи (в лекции — общий принцип «не раздувать выгрузку»).

#### 2.2 Почему `Пауза` не сработала из клиентского серверного вызова

**`Пауза`** в типовой модели допустима в контекстах **без прямого вызова с клиента**: фоновые задания, **веб‑/HTTP‑сервисы**, боты и т.п. (формулировка из синтакс-помощника/лекции).

При вызове **из формы** (клиент → сервер для отображения UI) пауза **запрещена** — платформа вернёт ошибку.

#### 2.3 «Варварская» задержка: цикл по времени

Альтернатива для учебной имитации: цикл, сравнивающий текущую дату/время сеанса с **«временем окончания задержки»**.

**Минусы:**

- **Грузит процессор** (активное ожидание), в отличие от **`Пауза`**, которая **не** расходует CPU тем же способом;
- на **8+ секунд** интерфейс «замёрз» — это именно то, чего нужно избегать в продакшене.

#### 2.4 Последствия для веб‑клиента

- Браузер может предложить **прервать** долгий скрипт / «сервер не отвечает».
- Прокси/шлюз может вернуть **504 Gateway Timeout** при превышении таймаута ответа.
- Пользователь воспринимает систему как **неработоспособную**.

**Вывод лекции:** «страдает» разработчик (фон, контроль ошибок), а не пользователь — долгое ожидание без индикации недопустимо.

### Блок 3: Модуль менеджера и подготовка к фону (8 минут)

#### 3.1 Перенос функции в менеджер документа

Функция вроде **«очень длительное и сложное заполнение»**:

- параметры: например **дата документа**, **длительность имитации** (секунды);
- внутри: **запрос** к справочнику **«Складские помещения»** (владелец — склад) с псевдонимами колонок, удобными для загрузки в ТЧ;
- результат: **таблица значений**, затем **`Товары.Загрузить(Результат)`** — внутри менеджера при прямом вызове (для отладки до ввода длительных операций).

Метод помечают **`Экспорт`**, чтобы на него можно было сослаться **строкой имени** из **`ДлительныеОперации.ВыполнитьФункцию`**.

### Блок 4: API длительных операций БСП (22 минуты)

#### 4.1 Серверная «обёртка»: начать выполнение

**Идея:** серверная функция формы (или общий серверный модуль) **не** вызывает тяжёлую функцию напрямую, а **регистрирует** её выполнение через БСП и возвращает **описание длительной операции** (структуру для мониторинга на клиенте).

**Шаги (логика из демо):**

1. **`ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(...)`**  
   В том числе передаётся **уникальный идентификатор формы** — чтобы при **закрытии** формы связанная длительная операция **корректно завершалась** (освобождение ресурсов; детали — в документации к вашей версии БСП).

2. **`РезультатДлительнойОперации = ДлительныеОперации.ВыполнитьФункцию(ПараметрыВыполнения, "Документы.РеализацияТоваровУслуг.ОченьДлительноеИСложноеЗаполнение", <параметры функции>...)`**  
   Строка метода — **полное имя экспортной функции** менеджера или общего модуля.

3. **`Возврат РезультатДлительнойОперации`** — на клиент.

#### 4.2 Клиент: оповещение и ожидание

1. **Создать описание оповещения** (`Новый ОписаниеОповещения`):  
   процедура **`ЗавершениеОченьДлительнойОперации`** на **клиенте**, **`Экспорт`** — иначе платформа **не сможет** вызвать обработчик из оповещения.

2. Параметры оповещения: обычно **модуль** (`ЭтотОбъект` для формы) и контекст.

3. **`ДлительныеОперацииКлиент.ОжидатьЗавершения(ДлительнаяОперация, ОповещениеОЗавершении, ПараметрыОжидания)`**  
   Связывает **запущенную** операцию и **callback** по завершении.

#### 4.3 Параметры ожидания и UX

**`ПараметрыОжидания = ДлительныеОперацииКлиент.ПараметрыОжидания(...)`** (имя метода в вашей редакции БСП уточнять по ИТС).

Настраивают:

- **владелец формы** — привязка немодального окна ожидания;
- **заголовок** и **текст** сообщения в окне ожидания («Выполняется получение данных…»);
- блок **оповещения пользователя** по завершении: флаг **«Показать»**, **текст** и **пояснение** (например: «Табличная часть заполнена»; при необходимости — **навигационная ссылка** на объект).

Пользователь в период ожидания **может** переключаться на другие задачи в той же базе (в отличие от синхронной блокировки).

#### 4.4 Обработчик завершения на клиенте

**Алгоритм:**

1. Если **`Результат = Неопределено`** (отмена и т.п.) — **`Возврат`**.

2. Проверить **статус** длительной операции на **ошибку** — вывести текст через типовые процедуры БСП (в демо копировали готовый фрагмент из шаблона / **`СтандартныеПодсистемыКлиент`** — точные имена методов по версии БСП).

3. При успехе из структуры результата взять **`АдресРезультата`** (временное хранилище). На **клиенте** нельзя напрямую «присвоить» таблицу значений в **`Объект.Товары`** из адреса — вызывают **`&НаСервере`** процедуру **`ЗаполнитьТоварыИзВременногоХранилища(Адрес)`**, которая делает **`Объект.Товары.Загрузить(ПолучитьИзВременногоХранилища(Адрес))`** (или аналог по API вашей версии).

### Блок 5: Резюме паттерна и что дальше (7 минут)

**Чек-лист разработчика:**

| Этап | Где | Что |
|------|-----|-----|
| Тяжёлая логика | Модуль **менеджера** (экспорт) | Запрос, формирование таблицы, помещение результата во **временное хранилище** (по правилам БСП) |
| Старт | Сервер: форма / модуль | **`ДлительныеОперации.ВыполнитьФункцию`** + параметры выполнения |
| UX | Клиент | **`ОжидатьЗавершения`** + **параметры ожидания** (окно + оповещение по завершении) |
| Финанш | Клиент → сервер | Разбор адреса, загрузка ТЧ на сервере |

**На следующих занятиях (по плану курса):** **многопоточное** выполнение, углубление в отложенные сценарии и смежные темы (в конце транскрипта — отсылка к **многопотокам**).

## Резюме лекции (5 минут)

### Ключевые моменты

1. Длительную **серверную** работу из UI нельзя выполнять **синхронно** — блокировка клиента, риски **таймаутов** в вебе, плохой UX.
2. Код заполнения — в **модуле менеджера** документа (переиспользование с формы и из кода).
3. **`Пауза`** — не из произвольного клиентского вызова; имитация циклом по времени **вредна** для CPU.
4. **`Выполнить().Выгрузить()`** на огромных выборках — риск по **памяти**; для high-load нужны иные приёмы чтения данных.
5. **`ДлительныеОперации.ПараметрыВыполненияФункции`** + **`ВыполнитьФункцию`** — запуск в фоне, возврат **структуры** для ожидания.
6. **`ДлительныеОперацииКлиент.ОжидатьЗавершения`** + **описание оповещения** (экспортная процедура на клиенте).
7. Результат — **адрес во временном хранилище**; перенос в **`Объект`** — отдельным **серверным** вызовом из обработчика завершения.

### Вопросы для самопроверки

1. Почему длительное заполнение табличной части не размещают в модуле формы?
2. Чем опасна полная **выгрузка** результата запроса в таблицу значений на высоконагруженных системах?
3. Почему **`Пауза`** может быть недоступна при вызове с формы?
4. Чем «цикл ожидания по текущей дате» хуже **`Пауза`** с точки зрения ресурсов?
5. Какие проблемы даёт **8+ секунд** синхронного вызова в **веб‑клиенте**?
6. Что такое **`ПараметрыВыполненияФункции`** и зачем передавать **идентификатор формы**?
7. Что возвращает **`ДлительныеОперации.ВыполнитьФункцию`**?
8. Почему процедура обработки оповещения на клиенте должна быть **`Экспорт`**?
9. Зачем нужны **`ПараметрыОжидания`** и оповещение пользователя по завершении?
10. Почему табличную часть из адреса результата заполняют **серверной** процедурой, а не прямо на клиенте?
