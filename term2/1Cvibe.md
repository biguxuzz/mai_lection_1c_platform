**Цель:** Познакомить студентов с концепцией RAG и ее применением

**План:**
- Что такое RAG и зачем он нужен

- Embedding модели и их выбор
- Chunking стратегии для документов
- **Графовые RAG системы (Graph RAG)**
  - Отличия от векторных RAG
  - Преимущества графового подхода для сложных связей
  - Knowledge graphs и entity relationships
- **LightRAG - современная реализация Graph RAG**
  - Архитектура LightRAG
  - Преимущества: низкие требования к ресурсам, высокая скорость
  - Сценарии использования для документации 1С

### [Лекция 8: RAG - практика и продвинутые техники](lection08/lection08.md)
**Цель:** Научить студентов решать практические проблемы RAG-систем, освоить GraphRAG с Neo4j и интегрировать RAG в реальные системы

**План лекции:**

1. **Повторение и решение проблемы с нулевыми векторами**
   - Краткое повторение основных концепций RAG из лекции 7
   - Проблема с нулевыми векторами в ноде n8n OpenAI Embedding
   - Демонстрация переключения на ноду Ollama Embedding
   - Практическое решение проблемы и проверка результатов

2. **Демонстрация семантического поиска**
   - Индексация документа `term1\lection02\lection02.md` (лекция об объектах хранения данных в 1С)
   - Примеры семантического поиска (не по ключевым словам):
     * Поиск по концепциям: "Как хранить условно постоянные значения?"
     * Поиск по связям: "Что такое подчиненные справочники и как они связаны с контрагентами?"
     * Поиск по назначению: "Какой объект использовать для фиксации операций?"
   - Сравнение семантического и ключевого поиска
   - Анализ качества результатов

3. **Проблемы векторного RAG и решение через GraphRAG**
   - Озвучивание ограничений векторного поиска:
     * Потеря контекста связей между сущностями
     * Сложность поиска по абстрактным концепциям
     * Проблемы с многошаговыми запросами
   - Введение в GraphDB:
     * Структура графовых баз данных (узлы, связи, свойства)
     * Преимущества для хранения знаний и связей
     * Назначение GraphDB для семантических данных
   - GraphRAG и LightRAG:
     * Концепция GraphRAG - поиск по графу знаний
     * LightRAG - легковесная реализация GraphRAG
     * Использование Neo4j для хранения графа знаний
     * Комбинация векторного поиска и графового анализа

4. **Практическая реализация LightRAG**
   - Развертывание LightRAG в Docker
   - Интеграция LightRAG в n8n workflow
   - Демонстрация работы LightRAG на примере лекции 2 прошлого семестра
   - Сравнение результатов векторного RAG и LightRAG

**Практическая часть:**
- Решение проблемы с нулевыми векторами в n8n
- Индексация и семантический поиск по лекции 2
- Настройка LightRAG в Docker
- Демонстрация полного workflow в n8n с LightRAG

## Курсовая работа: ИИ-агент в приложении 1С

### Возможные варианты тем:

1. **Агент-помощник разработчика**
   - Анализ кода и поиск ошибок
   - Генерация документации
   - Рефакторинг и оптимизация кода
   - Поиск по документации платформы

2. **Агент для работы с документами**
   - Интеллектуальная обработка входящих документов
   - Автоматическое заполнение реквизитов
   - Классификация и маршрутизация документов
   - Извлечение данных из неструктурированных источников

3. **Агент-аналитик**
   - Анализ данных из информационной базы
   - Генерация аналитических отчетов
   - Ответы на вопросы на естественном языке
   - Прогнозирование и рекомендации

4. **Агент технической поддержки**
   - Обработка запросов пользователей
   - Поиск решений в базе знаний
   - Генерация инструкций и справок
   - Автоматизация типовых задач поддержки

### Требования к курсовой работе:

**Обязательные компоненты:**
- Интеграция с LLM (локальной или API)
- Использование RAG или другой техники работы с контекстом
- Пользовательский интерфейс в 1С
- Система промптов
- Обработка ошибок и валидация
- Документация проекта

**Критерии оценки:**
- Работоспособность решения (40%)
- Качество кода и архитектуры (25%)
- Полнота функционала (20%)
- Документация и презентация (15%)

**Этапы сдачи:**
1. Выбор темы и согласование ТЗ (неделя 8)
2. Промежуточная защита - демонстрация прототипа (неделя 12)
3. Финальная защита с презентацией (неделя 16)
