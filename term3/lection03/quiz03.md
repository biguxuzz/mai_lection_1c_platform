# Quiz: Лекция 3 + семинар 3 — Длительные операции и Vanessa Automation

## Вопрос 1

**Почему логику массового заполнения документа рекомендуют выносить в модуль менеджера, а не оставлять только в модуле формы?**

1. Потому что модуль формы нельзя отладить
2. Потому что менеджер выполняется быстрее любого серверного вызова
3. Потому что код может понадобиться вызвать программно без открытия формы, а строка вида `Документы.Имя.Метод` удобна для вызова из других мест
4. Потому что БСП запрещает экспортные процедуры в форме

## Вопрос 2

**Чем опасен для сервера типовой приём `Запрос.Выполнить().Выгрузить()` на очень больших объёмах данных?**

1. Вся выборка может оказаться в оперативной памяти сразу, что раздувает потребление памяти
2. Платформа всегда режет выборку до 1000 строк без предупреждения
3. Метод `Выгрузить()` в новых версиях запрещён в запросах
4. Это влияет только на скорость печати, не на память

## Вопрос 7

**Почему результата длительной операции обрабатывают отдельной серверной процедурой, а не «напрямую» на клиенте?**

1. На клиенте нельзя работать с иммутабельными типами, такими как таблица значений
2. Потому что временное хранилище доступно только в толстом клиенте
3. Потому что БСП запрещает любые серверные вызовы после длительной операции
4. Потому что табличная часть хранится только в SQL

## Вопрос 8

**В чём суть режима отладки длительных операций в крупной конфигурации?**

1. Все фоновые задания отключаются навсегда
2. Отладка возможна только на файловой базе без сервера

4. Режим включает автоматическую запись поведения

## Вопрос 10

**Почему не рекомендуют делать менеджер тестирования на базе крупной промышленной ERP?**

1. Потому что ERP не поддерживает тонкий клиент
2. Лишний вес, лишние объекты и фоновая активность — плохая практика; целевая база — максимально лёгкая
3. Потому что в ERP нет ролей
4. Потому что Vanessa работает только с файловой базой размером до 1 МБ
