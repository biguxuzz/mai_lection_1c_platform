# Quiz: Семинар 2 — Базовая функциональность БСП (часть 2)

## Вопрос 5
**Чем плохо в цикле по строкам ТЧ обращение `СтрокаТоваров.Склад.ИспользуютсяСкладскиеПомещения` (через точку)?**

1. Такой синтаксис в 1С запрещён компилятором
2. Это всегда быстрее, чем запрос
3. Под капотом может загружаться **весь объект** справочника в память ради одного реквизита (верно)
4. Это ошибка только при работе в веб-клиенте

## Вопрос 6
**Что возвращает `ОбщегоНазначения.ЗначенияРеквизитовОбъектов` при передаче массива ссылок на склады и имени реквизита?**

1. Таблицу значений с колонками «Ссылка» и «Реквизит»
2. Всегда одно булево по первой ссылке
3. **Соответствие**: ключ — ссылка, значение — значение реквизита (в т.ч. булево флага помещений) (верно)
4. Массив структур без ключей

## Вопрос 7
**Чем для пользовательской ошибки заполнения в табличной части предпочтительнее связка ТекстОшибкиЗаполнения + СообщитьПользователю по сравнению с Сообщить?**

1. Сообщить всегда подставляет номер строки автоматически
2. Связка БСП позволяет оформить текст в стиле платформы и привязать переход к полю/колонке по ПутьКТабличнойЧасти (верно)
3. Сообщить запрещено в управляемых формах
4. Разницы нет, это вкусовщина

---

## Вопрос 9
**Почему вызов ЗначениеРеквизитаОбъекта внутри цикла по каждой строке табличной части может быть плохим решением?**

1. Метод работает только для документов, не для справочников
2. Получается запросы/вызовы в цикле, что противоречит стандартам минимизации таких обходов (верно)
3. Метод возвращает только строку, не числа и булевы
4. Циклы в 1С по табличным частям запрещены
