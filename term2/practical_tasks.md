## Задание 1: Работа с 1С Напарником - первая генерация кода

**Цель:** Освоить базовую работу с 1С Напарником для генерации кода в EDT.

**Задачи:**
1. Открыть EDT (Enterprise Development Tools для 1С)
2. Открыть или создать проект конфигурации 1С
3. Открыть модуль объекта (например, модуль формы документа) в редакторе
4. Открыть 1С Напарник (через меню или панель инструментов)
5. Написать простой промпт в 1С Напарнике: "Напиши процедуру на языке 1С, которая выводит сообщение 'Привет, мир!'"
6. Сгенерировать код и вставить его в модуль
7. Проверить синтаксис кода через встроенную проверку EDT
8. Попросить Напарника переписать код с использованием функции `Сообщить()` вместо `ВывестиТекстВКонсоль()`
9. Протестировать сгенерированный код

**Критерии оценки:**
- Успешный запуск EDT и открытие проекта
- Корректная работа с 1С Напарником в EDT
- Корректно сгенерированный и вставленный код
- Умение модифицировать запрос для улучшения результата
- Работающий код без синтаксических ошибок

## Задание 3: Установка и настройка VS Code с плагином Continue

**Цель:** Настроить рабочее окружение для вайб-кодинга с 1С в VS Code.

**Задачи:**
1. Установить VS Code (если еще не установлен)
2. Установить расширение BSL Extension Pack для 1С
3. Установить расширение Continue (open-source AI code agent)
4. Отключить подсветку Unicode символов (настройка `highlight: non-basic SC2 = false`)
5. Настроить Continue: добавить модель (локальную через LM Studio или облачную через API)
6. Выгрузить тестовую конфигурацию из 1С в файлы
7. Создать файл `.cursorignore` с содержимым `*.xml` (для совместимости с Continue)
8. Открыть проект в VS Code и проверить подсветку синтаксиса BSL
9. Протестировать работу Continue на простом примере генерации кода

**Критерии оценки:**
- Корректная установка всех компонентов (VS Code, BSL Extension Pack, Continue)
- Правильная настройка Continue с подключенной моделью
- Правильная настройка `.cursorignore`
- Работающая подсветка синтаксиса BSL
- Успешная генерация кода через Continue

## Задание 7: Работа с Docker: создание первого контейнера

**Цель:** Освоить базовые команды Docker и создать первый контейнер.

**Задачи:**
1. Установить Docker Desktop (если еще не установлен)
2. Проверить установку: `docker --version`
3. Скачать образ: `docker pull hello-world`
4. Запустить контейнер: `docker run hello-world`
5. Посмотреть список контейнеров: `docker ps -a`
6. Удалить контейнер: `docker rm <container_id>`

**Критерии оценки:**
- Успешная установка Docker
- Корректное выполнение базовых команд
- Понимание разницы между образом и контейнером

## Задание 8: Создание Docker Compose файла для MCP сервера

**Цель:** Научиться оркестрировать контейнеры с помощью Docker Compose.

**Задачи:**
1. Создать папку `mcp-server` для проекта
2. Создать файл `docker-compose.yml` со следующей структурой:
   - Сервис для MCP сервера
   - Проброс порта 8080:8080
   - Переменные окружения для настройки
   - Автозапуск при перезагрузке (`restart: unless-stopped`)
3. Создать файл `.env` для чувствительных данных
4. Запустить: `docker compose up -d`
5. Проверить работу через браузер или curl

**Критерии оценки:**
- Корректный синтаксис docker-compose.yml
- Правильное использование переменных окружения
- Успешный запуск сервиса

## Задание 15: Сравнение векторного RAG и GraphRAG

**Цель:** Понять различия между подходами и выбрать подходящий для задачи.

**Задачи:**
1. Взять один и тот же документ (например, лекцию про объекты 1С)
2. Проиндексировать его в векторную БД (Qdrant) - векторный RAG
3. Проиндексировать его в LightRAG - GraphRAG
4. Выполнить одинаковые запросы к обоим системам:
   - "Какой объект используется для хранения операций?"
   - "Как связаны документы и регистры?"
5. Сравнить результаты по:
   - Релевантности ответов
   - Учету связей между сущностями
   - Полноте контекста
6. Записать выводы о том, когда какой подход лучше использовать

**Критерии оценки:**
- Успешная настройка обоих подходов
- Корректное сравнение результатов
- Обоснованные выводы о применении
